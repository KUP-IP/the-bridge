### Security audit (v3.6 new-surface review)
- **Verified safe:** all v3.6 filesystem writes (`CommandStore`,
  `StandingOrdersStore`, `PathMigration` sentinel) use atomic-write
  primitives with `BridgePaths`-resolved URLs. No write paths accept
  external strings unsanitized.
- **Verified safe:** `CommandStore.delete/get/update` gate on the
  index (which only contains slugify-blessed keys), so path traversal
  via crafted slug strings is impossible at the API boundary.
- **Hardened:** `CommandStore.slugify` previously accepted Unicode `Ll`
  (lowercase-letter category), which permitted homoglyph collisions
  (Cyrillic `а` U+0430 vs ASCII `a`). Locked to ASCII `[a-z0-9_-]`.
  Two visually-identical command names can no longer produce different
  slugs and bypass the duplicate-slug check.
- **Verified safe:** PKT-877 dispatch fail-closed contract preserved.
  `BridgeToolError.moduleGroupDisabled` is the only path for disabled
  tools; no silent failure.

### Accessibility pass
- `BridgeSettingsSectionHeader` now provides combined a11y

  on the title and the decorative icon hidden). Every Settings section
  inherits the upgrade automatically.
- `AdvancedSection`: copy-URL and reveal-in-Finder icon-only buttons
  now have explicit accessibility labels.
- `IconPickerSheet`: color swatches expose `\(colorName) color` labels
  with `.isSelected` trait; search field labeled per active tab;
  decorative magnifying-glass icon hidden; clear-search labeled.
- `OnboardingWindow`: 813-line file left untouched in this packet —
  most of its controls already use text-labeled `Button("...")` which
  SwiftUI auto-labels. Decorative `arrow.up.right.square` icons are
  inline with text and inherit context. A deeper pass belongs in v3.7
  if VoiceOver users surface concrete gaps.
