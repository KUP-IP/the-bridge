# Acceptance Matrix — T1–T114 classification (Lane H)

**Source of truth:** PRD v1.0 §11 (T1–T114), §8.20, with controller-logic citations from §6 (FR-1..FR-26), §7 (cycle algorithm), §8.5 (receipt + receipt-to-status mapping), §8.8 (replay/idempotency), §8.10 (review/`PACKET DECISION`), §8.11 (cycle health), §8.14 (stale-state), §8.15 (compaction), §8.16 (Source of Truth), §8.17 (cleanup), §8.18 (preservation), §8.19 (archival), §8.21 (repo/code).

**Classes**
- **AUTOMATABLE-NOW** — the test's *core assertion* is a deterministic pure-function or state-machine decision over given inputs (receipt fields, status, timestamps, schema state, config, dependency graph) that is unit-testable with synthetic fixtures, **no live provider session and no authorized-human decision**. The exact deterministic check is named per row.
- **PROVIDER-INTEGRATION** — the assertion fundamentally requires a live provider routine actually dispatching/owning a worker, native execution URL/correlation, overlap guard, repo isolation, completion notification, or session export — observable only against a real provider + repo (§8.5A, FR-9, §11 intro "provider session and repository evidence").
- **GOVERNANCE** — the test subject is a real authorized-human decision artifact: a `PACKET DECISION` comment (§8.10), a cancellation, an operator re-enable (§8.13), or qualification sign-off (§8.20). Cannot be fabricated.
- **PILOT-ONLY** — T94–T98 and anything requiring the 20-cycle scheduled streak; "satisfied only by the real scheduled pilot, not simulation" (§11 intro, §8.20).

**Conservative rule applied:** if the headline scenario needs a live provider/human to *exist*, it is NOT automatable-now. Several rows are *hybrid* (a deterministic decision core wrapped by a live trigger). A row is marked AUTOMATABLE-NOW **only when the decision function is the true subject** and the provider/human part is mere fixture setup; the deterministic check is named, and the live dependency is noted in how-to-verify. Where the live artifact itself is the subject, the row is PROVIDER-INTEGRATION / GOVERNANCE / PILOT-ONLY and the (separately testable) deterministic sub-helper, if any, is noted but does **not** reclassify the row.

**Reading the table:** AUTOMATABLE-NOW rows have a deterministic decision as their true subject (named per row); they need no live provider and no human decision.
